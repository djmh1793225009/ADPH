# ADPH — Architecture Digital Prompt Hub


## 项目简介

本项目针对 AIGC（AI Generated Content）在建筑设计领域应用时存在的随机性高、文本控制力不足、难以维持结构稳定性等痛点，通过跨学科技术整合，构建了一套面向建筑行业的结构化提示词语料库（SOP），并配套开发了可视化交互展示系统。

**核心贡献：**

- **结构化语料库**：深入解构建筑专业语言，建立自然语言与建筑形式语言的精确映射，涵盖地域文化、物理材质、光影环境等维度，共覆盖 8 大风格类别。
- **精确空间约束**：利用 Canny（边缘检测）与 Depth（深度估算）ControlNet 模型锁定建筑几何拓扑，实现"同构异质"的稳定风格迁移。
- **本地化工作流**：基于 ComfyUI / Stable Diffusion 节点式框架，配合 LoRA 微调权重，优化显存分配，支持高效、可控的批量生产。
- **专业采纳建议**：为每种风格提供融合建筑设计与插画设计双重视角的专业选型建议，指导实际场景应用。
- **交互式前端展示**：基于 React + Vite + TypeScript 开发 Web 交互系统，通过动态切片展示 AIGC 辅助设计的逻辑与成果。

## 技术路线

```
原始提示词采集
      ↓
数据清洗 & 结构化（SOP v1.0 → v3.0）
      ↓
几何底模构建（Canny 轮廓 + Depth 深度图）
      ↓
ComfyUI 节点工作流搭建（SD + LoRA）
      ↓
多维度风格迁移渲染（8 大类别）
      ↓
采纳建议撰写 & 语料库完善
      ↓
Web 可视化交互展示（React + Vite）
```

1. **数据预处理**：对采集的提示词进行清洗降噪，剔除无效修饰词，从 `SOP_v1.0` 逐步迭代至 `SOP_v3.0` 结构化表格。
2. **几何底模构建**：选取现代综合体作为建筑原型，提取 Canny 轮廓线与 Depth 深度图，作为 ControlNet 的控制信号。
3. **节点工作流搭建**：在本地部署 Stable Diffusion 环境，接入特定建筑材质的 LoRA 微调权重，搭建可复用节点流。
4. **风格迁移实验**：在空间约束下，针对传统地域、现代材质、幻想概念进行多维度渲染测试，覆盖 8 大风格类别。
5. **采纳建议优化**：为每种风格撰写专业选型建议，融合建筑设计与插画设计双重视角。
6. **可视化开发**：基于 React 构建前端页面，实现同构异质切片的动态拼接展示。

## 快速开始

### 查阅提示词库

打开 `doc/SOP_latest.xlsx` 或 `doc/SOP_v3.0/SOP_v3.0.xlsx`，可直接获取针对各建筑风格优化的正向 / 反向提示词及采纳建议。

### 启动交互展示网站

**环境要求：** Node.js >= 18

```bash
cd app
npm install
npm run dev
```

启动后在浏览器访问 `http://localhost:3000`，即可浏览建筑风格图集并查看对应提示词数据。

## 结论与意义

本研究不仅产出了一套高质量的建筑插画风格图集，更重要的是跑通了"数据清洗 → 工作流搭建 → 空间约束渲染 → 前端交互展示"的全链路闭环。它证明了在数字化工具时代，创作者可以通过参数框架锁定空间逻辑，将 AI 转化为精准的渲染执行工具。
